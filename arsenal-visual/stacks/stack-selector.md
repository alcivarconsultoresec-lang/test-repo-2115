# Selector de Stack NOX FORGE

## Nivel 1: HTML + CSS + JS

Usar cuando:

- Demo rápida.
- Landing simple.
- Prospecto necesita ver algo hoy.
- No hay datos persistentes.

## Nivel 2: HTML + JSON interno

Usar cuando:

- Hay catálogo.
- Hay filtros.
- Hay lista de productos, emprendedores, servicios o paquetes.
- Se necesita actualización simple sin framework.

## Nivel 3: Vite + React + TypeScript

Usar cuando:

- Hay interfaz con varios componentes.
- Hay filtros avanzados.
- Hay estado interno.
- El proyecto se mantendrá a largo plazo.

## Nivel 4: Astro

Usar cuando:

- SEO importa.
- Página estática con contenido fuerte.
- Catálogo o blog con pequeñas islas interactivas.
- Se necesita velocidad alta.

## Nivel 5: Next.js

Usar cuando:

- Hay app completa.
- Hay rutas múltiples.
- Hay login.
- Hay APIs.
- Hay panel admin.
- Hay intención SaaS.

## Nivel 6: Datos persistentes

Opciones:

- Supabase.
- PostgreSQL.
- Firebase.
- Airtable.
- Google Sheets como CMS simple.

## Regla

No sobredimensionar. Un prospecto no necesita Next.js si una demo HTML resuelve la venta. Pero si el proyecto quiere crecer, elegir stack escalable desde el inicio.
