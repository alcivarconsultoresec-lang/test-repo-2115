# Matriz de conexiones MCP NOX FORGE

## Objetivo

Definir qué herramienta se usa en cada fase para convertir NOX FORGE en una navaja suiza de producción comercial y visual.

## Fase 1: Contexto comercial

Herramientas:

- Google Sheets: embudo, matriz de entregas, estados.
- Gmail: comunicaciones reales con prospectos.
- Google Calendar: reuniones, citas y seguimiento.
- Google Drive: documentos, propuestas, assets.

Uso:

- Identificar prospecto.
- Ver estado real.
- Detectar próxima acción.
- Evitar mensajes fuera de contexto.

## Fase 2: Investigación y arsenal

Herramientas:

- GitHub: protocolos, patrones, stacks, componentes.
- Web cuando se requiere información actual.
- Canva: referencias visuales y diseños existentes.

Uso:

- Consultar NOX FORGE.
- Seleccionar nicho.
- Revisar patrones.
- Preparar Design Canvas.

## Fase 3: Diseño visual

Herramientas:

- Canva: maqueta visual, piezas comerciales, Brand Kits.
- Canvas interno: diseño textual o prototipo de estructura.
- Image generation: imágenes conceptuales si se requieren.

Uso:

- Crear dirección visual.
- Validar estética antes de programar.
- Generar piezas de apoyo.

## Fase 4: Codificación

Herramientas:

- GitHub: crear/editar archivos.
- Repos mapeados: reutilizar patrones.
- Stack selector: decidir HTML, React, Astro o Next.

Uso:

- Crear demo.
- Versionar.
- Evitar destruir proyectos existentes.

## Fase 5: Publicación

Herramientas:

- Vercel: publicar, revisar deployment, logs y preview.
- GitHub Pages: revisión interna si aplica.

Uso:

- Validar enlace cliente.
- Revisar mobile.
- Verificar assets.

## Fase 6: Entrega comercial

Herramientas:

- Google Sheets: actualizar estado.
- Gmail/WhatsApp manual: preparar mensaje.
- Vercel: enlace final.

Uso:

- Crear mensaje de entrega.
- Añadir link directo.
- Registrar siguiente acción.

## Fase 7: Mejora continua

Herramientas:

- GitHub: documentar aprendizajes.
- Sheets: medir cierres y respuestas.
- Canva: mejorar identidad visual.
- Vercel: mantener versiones públicas.

Uso:

- Convertir cada proyecto en patrón reutilizable.
- Mejorar el arsenal.
- Reducir errores repetidos.

## Reglas operativas

- No programar antes de diseño aprobado.
- No entregar sin Vercel validado si es para cliente.
- No enviar mensajes sin revisar embudo.
- No modificar proyectos existentes sin leer archivo actual.
- No usar efectos sin propósito comercial o visual.
