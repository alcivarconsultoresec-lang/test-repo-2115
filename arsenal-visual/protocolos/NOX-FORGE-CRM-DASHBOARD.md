# NOX FORGE / CRM Dashboard

## Uso

Aplicar a CRMs, dashboards comerciales, centros de mando, paneles de seguimiento, embudos, reportes y herramientas internas.

## Objetivo

Crear sensación de software operativo, no de planilla decorada.

## Estructura obligatoria

1. Header command center.
2. KPIs principales.
3. Embudo visual.
4. Tabla o board de prospectos.
5. Filtros por estado, temperatura o prioridad.
6. Panel de acciones recomendadas.
7. Sección de alertas.
8. Historial o próximas acciones.

## Estética recomendada

Dark premium, tecnología, profundidad, bordes finos, brillo controlado y jerarquía clara.

## Paleta sugerida

- Fondo: #050B14
- Panel: #0D1726
- Panel elevado: #121F33
- Borde: #1F3A5F
- Cian: #00E5FF
- Verde: #21E6A5
- Alerta: #FFB020
- Texto: #F4F8FF

## Tipografías sugeridas

- Space Grotesk
- IBM Plex Sans
- Sora
- JetBrains Mono para métricas

## Componentes recomendados

- KPI cards.
- Funnel cards.
- Priority cards.
- Data table.
- Search input.
- Filter pills.
- Action queue.
- Status badges.
- Mini charts.

## Microinteracciones

- Hover en cards.
- Cambio visual de filtros.
- Barras animadas.
- Conteos dinámicos.
- Scroll suave.

## Errores a evitar

- Parecer Excel pegado.
- Falta de jerarquía.
- Demasiados colores sin sistema.
- Datos sin acción recomendada.
- Tabla sin filtros.

## Criterio de aprobación

El usuario debe sentir que está frente a una herramienta real para decidir qué hacer hoy.
